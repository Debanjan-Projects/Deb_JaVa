//Procedural Oriented Programming (POP):
In POP, the program is divided into small parts called functions or procedures, and the focus is on writing step-by-step instructions to solve a problem. It follows a top-down approach, where the main problem is broken into sub-problems and solved using functions. Data in POP is often shared globally and can be accessed by any function, which makes it less secure. Reusability of code is limited, as functions can only be reused by calling or copying them. Popular POP languages include C, Fortran, and Pascal.

Object-Oriented Programming (OOP):
In OOP, the program is divided into small units called objects, which contain both data and methods to operate on that data. It follows a bottom-up approach, focusing on creating reusable, modular, and secure code. Data is encapsulated within objects, and access is controlled, making OOP more secure. It also supports features like inheritance, polymorphism, and abstraction, which improve reusability and flexibility of code. Popular OOP languages include C++, Java, Python, and C#.

class Car {
    String model;
    int year;

    Car(String m, int y) {
        model = m;
        year = y;
    }

    // Copy constructor
    Car(Car obj) {
        model = obj.model;
        year = obj.year;
    }

    void display() {
        System.out.println("Model: " + model + ", Year: " + year);
    }

    public static void main(String[] args) {
        Car c1 = new Car("Audi", 2024);
        Car c2 = new Car(c1); // Copy constructor used
        c2.display();
    }
}

1. Primitive Data Types

Primitive data types are the basic or fundamental data types provided by Java.
They are predefined by the Java language and used to store simple values such as numbers, characters, or boolean values.

Examples:
byte, short, int, long, float, double, char, boolean

Characteristics:

Stored directly in memory where the variable is allocated.

Contain only a single value (no methods or properties).

Have a fixed size (e.g., int is always 4 bytes).

Faster in execution and memory-efficient.

Default values: 0 for numbers, false for boolean, ‚Äò\u0000‚Äô for char.

2. Non-Primitive Data Types

Non-primitive (or reference) data types are created by programmers and are used to store complex objects or multiple values.

Examples:
String, Array, Class, Interface, Object

Characteristics:

Store reference (memory address) of the actual object, not the object itself.

Can have methods and properties.

Variable size, depends on the object or data stored.

Slower than primitive types due to reference handling.

Default value: null (means no reference assigned).

2.Defaults Values: primitives have specific defaults like 0 or false reference types default to nuull.

3.speed: Access to primitive is generally faster.

4.Storage Location: primitive are stored in the stack , reference types are store in the heap .

5. comparison:Primitive compared by value:reference types compared by reference ..

1.Automatic process: garbage collection is managed by the java virtual machine(jvm), running in the background.

2.object Eligibility: objects that are no longer reachable meaning no active references to them , are eligible for grabage collection.

3.No manual control: Unlike languages like c++ , java developers cannot explicitly deallocate memory , Garbage collection is automatic and non-deterministic.

4.Generational collection: java uses a generational garbage collection strategy , which divides memory into different regions (young , old , and parmanent generations)based on objects ages.

5.heap memory : garbage collection occurs in the heap memory , where all java objects , reside.

//üîÅ 1Ô∏è‚É£ while loop
Definition:

The while loop is an entry-controlled loop ‚Äî meaning it checks the condition before executing the body of the loop.

Syntax:
while (condition) {
    // statements
}

How it works:

The condition is evaluated first.

If the condition is true, the body executes.

After execution, the condition is checked again.

Loop continues until the condition becomes false.

If the condition is false initially, the loop body never executes.

Use Case:

Used when the number of iterations is not known in advance (e.g., reading input until user stops).

Example:
int i = 1;
while (i <= 5) {
    System.out.println(i);
    i++;
}


**********************************
üîÇ 2Ô∏è‚É£ for loop
Definition:

The for loop is also an entry-controlled loop, but it is best used when the number of iterations is known in advance.

Syntax:
for (initialization; condition; update) {
    // statements
}

How it works:

Initialization runs once at the start.

Condition is checked ‚Üí if true, loop body executes.

After each iteration, update (like i++) runs.

The loop continues until condition becomes false.

Use Case:

Used for count-controlled loops (like printing numbers 1 to 100).

Example:
for (int i = 1; i <= 5; i++) {
    System.out.println(i);
}


*********************************
üîÑ 3Ô∏è‚É£ do-while loop
Definition:

The do-while loop is a exit-controlled loop ‚Äî meaning the condition is checked after executing the loop body.

Syntax:
do {
    // statements
} while (condition);

How it works:

The body executes at least once, even if the condition is false.

After executing the body, condition is checked.

If the condition is true, it repeats.

Use Case:

Used when you need the loop to run at least once, regardless of condition (like a menu-driven program).

Example:
int i = 1;
do {
    System.out.println(i);
    i++;
} while (i <= 5);

**********************************
üîÅ 4Ô∏è‚É£ for-each loop (enhanced for loop)
Definition:

The for-each loop is used to iterate over arrays or collections directly, without using an index variable.

Syntax:
for (dataType variable : arrayOrCollection) {
    // statements
}

How it works:

It automatically picks each element from the collection or array.

No need for manual index or counter.

It works only in forward direction (cannot modify or access index directly).

Use Case:

Used to read or display each element of an array or collection.

Example:
int[] numbers = {1, 2, 3, 4, 5};
for (int num : numbers) {
    System.out.println(num);
}
