//Procedural Oriented Programming (POP):
In POP, the program is divided into small parts called functions or procedures, and the focus is on writing step-by-step instructions to solve a problem. It follows a top-down approach, where the main problem is broken into sub-problems and solved using functions. Data in POP is often shared globally and can be accessed by any function, which makes it less secure. Reusability of code is limited, as functions can only be reused by calling or copying them. Popular POP languages include C, Fortran, and Pascal.

//ehar hum log function banati hain .
// each function have own local data and .
// also have some global data ..
//and global data must be accessable..



Object-Oriented Programming (OOP):
In OOP, the program is divided into small units called objects, which contain both data and methods to operate on that data. It follows a bottom-up approach, focusing on creating reusable, modular, and secure code. Data is encapsulated within objects, and access is controlled, making OOP more secure. It also supports features like inheritance, polymorphism, and abstraction, which improve reusability and flexibility of code. Popular OOP languages include C++, Java, Python, and C#.

6.10 Garbage collection and finalize..

1.Automatic process: garbage collection is managed by the java virtual machine(jvm), running in the background.

2.object Eligibility: objects that are no longer reachable meaning no active references to them , are eligible for grabage collection.

3.No manual control: Unlike languages like c++ , java developers cannot explicitly deallocate memory , Garbage collection is automatic and non-deterministic.

4.Generational collection: java uses a generational garbage collection strategy , which divides memory into different regions (young , old , and parmanent generations)based on objects ages.

5.heap memory : garbage collection occurs in the heap memory , where all java objects , reside.

6.performance performace: particularly if it runs frequently , or takes a  long time to complete .
